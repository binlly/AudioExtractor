# Release Notes - ExtractAudio v2.0.0

**发布日期**: 2026-03-12
**版本**: 2.0.0
**类型**: 主要版本更新

---

## 🎉 概述

ExtractAudio v2.0.0 是一次重大更新，引入了全新的视频预览播放器和可视化时间范围选择功能，彻底改变了音频提取的体验。用户现在可以在提取音频之前预览视频内容，并精确选择要提取的音频片段。

---

## ✨ 新功能

### 1. 视频预览播放器 🎬

- **内置视频播放**：直接在应用中播放视频，无需外部播放器
- **点击播放/暂停**：直观的触摸控制
- **实时进度显示**：显示当前播放位置和总时长
- **自动停止**：播放到结束时间时自动暂停并跳回开始位置

**使用场景**：
- 预览视频内容，确认要提取的音频片段
- 精确定位音频的关键时刻（如歌曲副歌、对话片段）

### 2. 可视化时间范围选择 🎚️

- **双滑块进度条**：
  - 开始滑块：控制音频提取的起始时间
  - 结束滑块：控制音频提取的结束时间
  - 选中区域高亮显示，一目了然

- **智能约束**：
  - 自动强制 开始时间 < 结束时间
  - 最小间隔 100ms，防止无效选择
  - 无法拖动滑块超出视频总时长

- **实时预览**：
  - 拖动滑块时自动暂停播放
  - 视频画面实时预览（节流 200ms）
  - 平滑的视觉反馈

### 3. 精确时间输入 ⏱️

- **毫秒级精度**：支持 `HH:MM:SS.mmm` 格式输入
- **双向同步**：
  - 拖动滑块 → 输入框实时更新
  - 修改输入框 → 滑块跳到新位置

- **智能验证**：
  - 实时格式验证
  - 非法输入显示橙色边框
  - 超出范围自动修正

- **持续时间显示**：自动计算并显示选中的时长

### 4. 智能文件命名 📝

提取的音频文件名自动包含时间范围信息，便于识别：

```
完整视频：
├── music_video.mp4 → music_video.mp3

片段提取：
├── concert.mp4 (00:01:23.456 - 00:05:30.789)
│   └── concert_00-01-23-456-00-05-30-789.mp3
```

**命名规则**：
- 完整视频：无后缀
- 片段提取：`_开始时间-结束时间` 后缀
- 冒号和点替换为连字符，兼容所有文件系统

## 🚀 性能改进

### FFmpeg 参数优化

**问题**：v1.0 使用 `-to` 参数指定结束时间，但 FFmpeg 需要相对时间

**解决方案**：
```bash
# v1.0 (错误)
ffmpeg -ss 00:01:23.456 -i input.mp4 -to 00:05:30.789 ...

# v2.0 (正确)
ffmpeg -ss 00:01:23.456 -i input.mp4 -t 00:04:07.333 ...
```

**改进**：
- ✅ 使用 `-t` 指定持续时间（结束时间 - 开始时间）
- ✅ 提取时长更准确
- ✅ 进度显示更精确

### 组件架构优化

- **VideoPlayerProvider**：独立的视频播放器状态管理
- **解耦设计**：视频播放和音频提取逻辑分离
- **更好的性能**：减少不必要的状态更新

## 🐛 Bug 修复

### 1. 时间范围提取时长不准确

**问题描述**：提取的音频时长与选择的时间范围不一致

**根本原因**：FFmpeg `-to` 参数的行为与预期不符

**修复方案**：改用 `-t` 参数计算持续时间

### 2. 进度显示不准确

**问题描述**：片段提取时进度百分比计算错误

**修复方案**：基于片段时长而非视频总时长计算进度

### 3. 文件拖拽响应延迟

**问题描述**：拖拽文件后加载提示显示不及时

**修复方案**：优化拖拽事件处理流程

## 📦 技术变更

### 新增依赖

```yaml
dependencies:
  video_player: ^2.8.0        # 视频播放支持
  desktop_drop: ^0.7.0        # 拖拽功能增强
```

### 新增组件

- `VideoPlayerWidget`：视频播放器组件
- `DualSliderProgressBar`：双滑块进度条
- `TimeRangeInputs`：精确时间输入框
- `VideoPlayerProvider`：视频播放器状态管理

### 移除组件

- `DropZone`：功能已集成到 VideoPlayerWidget
- `TimeRangeSelector`：被 TimeRangeInputs 替代

### 架构改进

```
v1.0 架构：
UI → ExtractionProvider → AudioExtractor → FFmpeg

v2.0 架构：
UI → VideoPlayerProvider + ExtractionProvider
                           ↓
                    AudioExtractor → FFmpeg
```

## ⚠️ 已知问题

### 1. 大视频文件加载时间较长

**影响**：>1GB 的视频文件可能需要 5-10 秒加载

**临时方案**：等待加载完成，或使用较小的测试视频

**计划修复**：v2.1 将优化大文件的初始化

### 2. 某些编码格式不支持预览

**影响**：部分 HEVC/VP9 编码的视频可能无法播放

**替代方案**：仍可提取音频，只是无法预览

**计划修复**：未来版本将添加更多解码器支持
